# Размещение глифов планет на зодиакальном кольце
## Референс для реализации в Transit Chart

---

### Проблема

Когда несколько планет находятся в узком диапазоне долгот, их глифы перекрывают
друг друга. Нужна стратегия размещения, которая сохраняет читаемость карты.

---

### Выбранное решение: двухуровневое радиальное смещение

Планеты размещаются на **двух концентрических «этажах»** внутри зодиакального
кольца. Выбор этажа зависит от размера кластера:

```
[ Zodiac outer ring  ─────────────────── ]   R_ZODIAC_OUT = 210
[ Level 1 (outer glyph track) ────────── ]   R_OUTER      = 220  ← 3+ планет
[ Level 0 (inner glyph track) ────────── ]   R_INNER      = 198  ← все планеты
[ Ecliptic tick dots ────────────────────]   R_TICK       = 182  ← истинная позиция
[ Zodiac inner ring  ─────────────────── ]   R_ZODIAC_IN  = 185
```

На `R_TICK` рисуется маленькая точка — истинная позиция планеты на эклиптике.
Если глиф смещён на уровень 1, точка соединяется с глифом тонкой линией.

---

### Алгоритм

**Шаг 1. Найти кластеры**

Сортируем планеты по долготе. Планеты, расстояние между которыми меньше
`min_gap` (обычно 6°), объединяются в один кластер. Учитываем переход через 0°.

**Шаг 2. Назначить уровни**

- Кластер **1–2 планеты** → все получают `level = 0`
- Кластер **3+ планет** → жадное распределение:
  - Идём по планетам кластера в порядке возрастания долготы
  - Пробуем поставить на уровень 0; если не умещается — на уровень 1
  - Если оба заняты — берём тот, где больше места

**Шаг 3. Вычислить координаты**

```python
angle = math.radians(180 - lon)          # 0° Овна = левая точка, CCW
r     = R_INNER if level == 0 else R_OUTER
x     = cx + r * math.cos(angle)
y     = cy - r * math.sin(angle)
```

---

### Параметры (подбирать под размер карты)

| Параметр  | Типичное значение | Описание                        |
|-----------|------------------:|---------------------------------|
| `min_gap` |              6.0° | Минимальный угловой зазор между глифами |
| `R_INNER` |          198–205  | Радиус первого этажа            |
| `R_OUTER` |          218–228  | Радиус второго этажа            |
| `R_TICK`  |          178–185  | Радиус точек истинной позиции   |

Разница `R_OUTER − R_INNER` ≈ 20–30px — «высота этажа».

---

### Что НЕ делает этот алгоритм

- Не двигает глифы по угловой оси (нет фан-аута).
- Не обрабатывает 3 уровня — только 0 и 1.
- Не адаптируется к размеру шрифта глифа автоматически.

Реализацию с визуальным тестом (перетаскивание + рандомайзер) — см. `glyph-levels.jsx`.
Питоновский код с `assign_levels()` и примером рендеринга — см. `glyph_levels.py`.
